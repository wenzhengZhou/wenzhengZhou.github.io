自己搭的梯子就是快

在vultr上买个了vps, 正好在上面搭个梯子玩玩,那些免费的梯子时好时坏,不稳定.

服务端安装

系统: Centos6.7

  1. 将python 升级到 python2.7
  2. 安装setuptools
  3. 安装pip, pip 类似 redhat里的yum
    wget “https://pypi.python.org/…….”
    tar -zxvf pip
    cd pip
    python setup.py install
  4. 安装相关组建
    yum install build-essential autoconf libtool openssl-devel gcc -y

  5. 安装git, 下载shadowsocks, 安装
    yum install git -y
    git clone https://github.com/madeye/shadowsocks-libev.git
    cd shadowsocks-libev
    ./configure
    make && make install

    配置config.json
    {
    “server”: “server ip”,
    “server_port”: port,
    “password”:”password”,
    “timeout”:60,
    “method”:”rc4-md5”
    }

    运行
    ss-server -c ~/config.json -f /tmp/ss.pid

这样服务端就搭好了, 下面配置客户端

ss-qt5, chrome+SwitchySharp+gfwlist
配置起来很简单,就不多写了

supervisor 托管 ss

  1. 安装
    pip install supervisor
  2. 初始化
    echo_supervisord_conf > /etc/supervisord.conf
    vim /etc/suervisord.conf
    在后面加入
    [program:shadowsocks]
    command = ss-server -c /home/zc/config.json
    user = zc
    autostart = true
    autorestart = true
    stderr_logfile = /var/log/supervisor/ss.stderr.log
    stdout_logfile = /var/log/supervisor/ss.stdout.log

  3. 运行supervisor
    supervisord -c /etc/suervisord.conf

    supervisor相关控制命令
    supervisorctl start all
    supervisorctl stop all
    supervisorctl restart all

    supervisorctl start shadowsocks

  4. 设为开机启动
    vi /etc/rc.local
    加入 supervisord -c /etc/suervisord.conf

遇到的问题

  1. pip安装第三方模块时提示 urllib3 的 SSL 连接失败错误
    yum install python-devel libffi-devel openssl-devel
    pip install pyopenssl ndg-httpsclient pyasn1

  2. 如果ss client连不上,看看是不是防火墙没设好
    可以用telnet 检查相应的端口是否能连接成功
