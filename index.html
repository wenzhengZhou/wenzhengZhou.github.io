Centos6.7 gfirefly搭建

  • 升级 python 到python2.7
    Twisted需要python2.7以上的版本

  • Install mysql
    yum install -y mysql-server mysql mysql-devel

  • Install setuptools
    见https://pypi.python.org/pypi/setuptools说明
    wget https://bootstrap.pypa.io/ze_setup.py -O - | sudo python

  • Install git
    yum install -y git

  • Download gfirefly
    git clone https://github.com/9miao/G-Firefly.git

  • Install gfirefly
    cd G-Firefly/gfirefly
    sudo python setup.py build
    sudo python setup.py install

  • 安装成功,创建示例
    python gfirefly/script/gfirefly-admin.py createproject ~/mygamesvr
    cd ~/mygamesvr
    sudo python startmaster.py

    打开别一个终端
    cd ~/mygamesvr/tool
    sudo python clienttest.py

    服务器正确关闭方法
    http://ip:port/stop

遇到的问题

  1. 安装MySQL-python 时出现错误 error: Setup script exited with error: command ‘gcc’ failed with exit status1
    gcc已安装,应该是某个依赖库没装
    yum install mysql-devel

  2. ImportError: Twisted requires Python 2.7 or later
    升级python 到 python2.7

  3. setup.py 安装Twisted错误, error: Not a recognized archive type: /tmp/easy_install…….
    手动安装Twisted
    wget 错误信息上面有地址
    tar -jxvf Twisted.tar.bz2
    cd Twisted
    sudo python setup.py install

  4. ZMQ错误,
    手动安装zeromq
    wget http://download.zeromq.org/zeromq-2.2.0.tar.gz
    tar -zxvf zeromq
    cd zeromq
    ./configure
    make
    sudo make install

    出现的错误 configure: error: cannot link with -luuid, install uuid-dev
    yum install uuid-devel
    yum install libuuid-devel

  5. 运行gfirefly实例,日志输出
    node [0] lose
    node [1] lose

    需要将gevent的版本换成1.0.2
    wget –no-check-certificate https://pypi.python.org/packages/source/g/gevent/gevent-1.0.2.tar.gz#md5=117f135d57ca7416203fba3720bf71c1
    tar -zxvf gevent
    cd gevent
    sudo python setup.py install

  6. 安装easy_install报错”urllib2.URLError:“
    yum install openssl-devel

  7. python-eggs错误
    perhaps your account does not have write access to this directory? you can change the cache directory by setting the PYTHON_EGG_CACHE environment variable to point to an accessible directory
    mkdir ~/.python-eggs
    mkdir /tmp/python-eggs
    export PYTHON_EGG_CACHE /tmp/python-eggs

centos6.7 python2.6升级到python2.7

  • 查看当前python版本
    python -V
    centos6.7 自带了python2.6

  • 下载安装
    wget https://www.python.org/ftp/python/2.7.11/Python-2.7.11.tgz
    tar -zxvf Python-2.7.11.tgz
    cd Python-2.7.11
    ./configure
    make all
    make install
    make clean
    make distclean

  • 建立软连接
    ln -s /usr/local/bin/python2.7 /usr/bin/python

  • 解决系统相关的兼容问题
    1 yum问题
    vim /usr/bin/yum
    将文件头部的 #!/usr/bin/python 改为 #!/usr/bin/python2.6

    2 iBus输入法问题
    打开 /usr/bin/ibus-setup, /usr/libexec/ibus-ui-gtk, /usr/libexec/ibus-engine-table 三个文件
    将文件中的所有python 换为 python2.6

狮子老板

狮子让一只豹子管理10只狼, 并给他们分发食物. 豹子领到肉之后, 把肉平均分成了11份, 自己要了一份, 其他给了10只狼. 这10只狼都感觉自己分的少, 合起伙来跟豹子唱对台戏. 虽然一只狼打不过豹子, 但10只狼豹子却没法应付了, 豹子灰溜溜的找狮子辞职. 狮子说, 看我的. 狮子把肉分成了11份, 大小不一, 自己先挑最大的一份, 然后傲然对其他狼说：你们自己讨论这些肉怎么分. 为了争夺到大点的肉，狼群沸腾了，恶狠狠的互相攻击，全然不顾自己连平均的那点肉都没拿到. 豹子钦佩的问狮子, 这是什么办法? 狮子微微一笑, 听说过绩效工资吗？……

第二天, 狮子依然把肉分成11块, 自己却挑走了2块, 然后傲然对其他狼说：你们自己讨论这些肉怎么分. 10只狼看了看9块肉, 飞快的抢夺起来, 一口肉, 一口曾经的同伴, 直到最后留下一只弱小的狼倒在地上奄奄一息.

centos6.7 安装python3

  • 下载python
    wget http://www.python.org/ftp/python/3.5.1/Python-3.5.1.tgz

  • 安装环境
    yum install gcc
    yum install zlib-devel
    yum install make
    yum install readline-devel

  • 解压,编译,安装
    tar -zxvf Python-3.5.1.tgz
    cd Python-3.5.1
    ./configure -prefix=/opt/python3
    make all
    make install

    清除编译出的可执行文件和配置文件
    make clean
    清除所有生成文件
    make distclean

  • 查看python3版本
    /opt/python3/bin/python3 –version

  • 做个软连接到当前用户的bin目录
    ln -s /opt/python3/bin/python3 ~/bin/python3

  • 安装好新版本python后,默认的依然是python2.6
    将文件头部的#!/usr/bin/python 修改为 #!/opt/python3/bin/python3 即可

遇到的问题

  • SecureCRT环境下 python3出现按键映射错误问题
    按backspace, 方向键时会出现^H^[[C之类的乱码
    原因: python是自己编译的, 在编译时配置脚本没有检测到readline开发文件, 所以它没有编译readline模块,
    解决方法: yum install readline-devel 然后重新编译安装python即可
